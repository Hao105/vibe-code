# 🛸 離線環境部署指南 (Windows 原生執行篇)

我們已經為您打包出了一包特殊的「快遞母艦」映像檔。這個映像檔的功能**不是為了長期待在 Docker 裡運行**，而是作為合規的運送載具，將為 Windows 編譯好的原生執行檔「卸載」到您的離線電腦上。

這樣做能完美保留每位連線者的**真實 IP** (因為不再透過 Docker NAT 代理)，同時又符合只能帶著 `.tar` 進入離線環境的資安流程。

---

## 🏗️ 第一步：把映像檔帶上車

專案根目錄中現在已經有這個檔案：
👉 **`vibe-code-windows-release-v5.tar`**

請把這個檔案複製到隨身碟上，帶往您的離線環境。

---

## 🚀 第二步：在目標電腦上「卸載」

抵達離線實體電腦後，請將 `.tar` 放進一個專門的資料夾中（例如：`D:\MyChat\`）。

1. **載入映像檔**：
   打開終端機或 PowerShell，進入該資料夾，並輸入：
   ```powershell
   docker load -i vibe-code-windows-release-v5.tar
   ```

2. **啟動自動卸載程序 (將產物倒出來)**：
   執行以下指令（此處以 `D:\MyChat` 為例，您可以換成任何您目前的資料夾，路徑請用**反斜線 `/`** 或**兩條斜線 `\\`** 防止跳脫）：
   ```powershell
   // 注意：路徑請用斜線 (/) 代替
   docker run --rm -v D:/MyChat:/host vibe-code-windows-release
   ```
   > 畫面會自動打印：*「🚚 正在將檔案卸載至您的 Windows 主機... ✅ 卸載完成！」*

3. **大功告成，您現在可以把 Docker 關掉了！**
   您會驚喜地發現 `D:\MyChat` 資料夾內憑空出現了：
   - 剛出爐的 `server.exe` (全新的主程式)
   - `dist/` (您的 React 前端網頁資料夾)
   - `whitelist.json` (白名單)
   - `cert.pem` 與 `key.pem` (安全憑證)

---

## ⚡ 第三步：啟動您的終極伺服器

以後每次開機，除了按照需求隨時使用記事本修改 `whitelist.json` 外，**您只需直接對著那個 `server.exe` 點兩下！** 

這支強化過的原生 Go 程式，現在會自動包辦所有事情：**不僅能處理聊天通訊，還兼顧提供前端網頁檔案 (`dist/`)。**

您的朋友只要在區域網路內開啟瀏覽器，輸入：
`https://（這台電腦的實體IP）:1501/vibe-code/`

就能順利看見精美的畫面，而且在不需要經過任何 Nginx 或 Docker NAT 的干擾下，您的白名單將能精確抓到並識別他們原汁原味的每一台**真實 IP**！
